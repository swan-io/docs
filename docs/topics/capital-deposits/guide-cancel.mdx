# Cancel a capital deposit case

## Guide {#steps}

1. Call the `cancelCapitalDepositCase` mutation.
1. Add the capital deposit case ID (line 2).
1. Add the success payload and any information you'd like to receive (line 3).
1. Add rejections (not shown).

## Payload {#payload}

The payload confirms the capital deposit case status changed to `Canceled`.

```json {6-9} showLineNumbers
{
  "data": {
    "cancelCapitalDepositCase": {
      "__typename": "CancelCapitalDepositCaseSuccessPayload",
      "capitalDepositCase": {
        "statusInfo": {
          "status": "Canceled",
          "reasonCode": "KYCIssue"
        }
      }
    }
  }
}
```

## What happens after cancellation {#after-cancellation}

When you cancel a capital deposit case, any associated shareholder accounts are automatically closed.
Starting 2 April 2026, these accounts receive the [closure reason code](../accounts/closure/index.mdx#closure-reason-codes) `CapitalDepositCanceled`.
Before this date, the closure reason code is `CapitalDepositReason`.

The [cancelation reason code](./index.mdx#cancelation-reason-codes) is also available directly on the capital deposit case via the `CapitalDepositCaseCanceledStatusInfo` type.

:::info Completed capital deposits
When a capital deposit is successfully completed, shareholder accounts are closed with the reason code `CapitalDepositCompleted`.
Before 2 April 2026, the closure reason code is `CapitalDepositReason`.
:::
