## API updates

### Upcoming breaking changes

üîç Starting **November 12th**:

- The [`onboarding(id: ID!)` query](https://api-reference.swan.io/queries/onboarding) may return `null` when no onboarding exists for the given ID, instead of always returning an [`Onboarding` object](https://api-reference.swan.io/objects/onboarding).
- The following deprecated fields will be removed:
  - **`verificationFlow`** from the [`OnboardingInfo` object](https://api-reference.swan.io/objects/onboarding-info/). 
  - **`redirectUrl`** from both the [`Onboarding`](https://api-reference.swan.io/objects/onboarding) and [`OnboardingInfo`](https://api-reference.swan.io/objects/onboarding-info) objects. It has been replaced by the [`oauthRedirectParameters` object](https://api-reference.swan.io/objects/oauth-redirect-parameters/).
  - **`title`** from the [Ultimate Beneficial Owner (UBO) object](https://api-reference.swan.io/objects/individual-ultimate-beneficial-owner/). It has been replaced by the [`gender` field](https://api-reference.swan.io/objects/individual-ultimate-beneficial-owner#individualultimatebeneficialownergendergender-).
  - **`onboardingState`** from the [`OnboardingInfo` object](https://api-reference.swan.io/objects/onboarding-info/).

üí∏ On **October 19th**, we will introduce the following updates to the `initiateCreditTransfers` endpoint to improve its usability and clarity:

- The `save` parameter now defaults to `false`.
- The `mode` parameter is now a **required** field. The default value is set to `Regular`.
- The `isMyOwnIban` and `beneficiaryId` input fields have been **removed**.
- The API now returns a `ValidationRejection` error instead of `ForbiddenRejection` when an input contains **invalid** data.
