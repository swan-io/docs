# Delete a document using the API

Delete a document you or your end users uploaded, with the API.

:::tip Prerequisites
You'll need either a [project access token](/build/using-api/authentication#tokens-project), or a [user access token](/build/using-api/authentication#tokens-user) for the associated user.
Also, the [document must have the status](/accounts/concepts/documents/statuses#document-statuses) `Uploaded`.
:::

## Guide {#guide}

1. First, [retrieve the document ID](/accounts/guides/documents/get-info).
1. Call the `deleteSupportingDocument` mutation.
1. Enter the document ID retrieved in step 1.
1. Add optional messages to the success payload, either for validation or in case of rejection.

### Mutation

<a href="https://explorer.swan.io/?query=bXV0YXRpb24gRGVsZXRlRG9jdW1lbnQgewogIGRlbGV0ZVN1cHBvcnRpbmdEb2N1bWVudChpbnB1dDogeyBpZDogIiRET0NVTUVOVF9JRCIgfSkgewogICAgLi4uIG9uIERlbGV0ZVN1cHBvcnRpbmdEb2N1bWVudFN1Y2Nlc3NQYXlsb2FkIHsKICAgICAgaWQKICAgIH0KICB9Cn0K&tab=api" className="explorer-badge">Open in API Explorer</a>

```graphql {2} showLineNumbers
mutation DeleteDocument {
  deleteSupportingDocument(input: { id: "$DOCUMENT_ID" }) {
    ... on DeleteSupportingDocumentSuccessPayload {
      id
    }
  }
}
```

### Payload

If the deletion is successful, the API Explorer returns the document ID.
You can confirm that the document no longer exists by checking your Dashboard.

```json {4} showLineNumbers
{
  "data": {
    "deleteSupportingDocument": {
      "id": "f193d0f6-fe7e-4592-9b5f-bce46a968553"
    }
  }
}
```
