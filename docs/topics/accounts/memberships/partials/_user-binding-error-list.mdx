At least one of the following booleans must be true for an account membership to have a user binding error:

<table>
  <tr>
    <th>Error</th>
    <th>If <code>true</code></th>
    <th>Update</th>
  </tr>
  <tr>
    <td><code>firstNameMatchError</code></td>
    <td>There's a mismatch with the first or given name.</td>
    <td>Update <code>restrictedTo</code> > <code>firstName</code>.</td>
  </tr>
  <tr>
    <td><code>lastNameMatchError</code></td>
    <td>There's a mismatch with the last or family name.</td>
    <td>Update <code>restrictedTo</code> > <code>lastName</code>.</td>
  </tr>
  <tr>
    <td><code>birthDateMatchError</code></td>
    <td>There's a mismatch with the birth date.</td>
    <td>Update <code>restrictedTo</code> > <code>birthDate</code>.</td>
  </tr>
  <tr>
    <td><code>mobilePhoneMatchError</code></td>
    <td>There's a mismatch with the phone number.</td>
    <td>Update <code>restrictedTo</code> > <code>phoneNumber</code>.</td>
  </tr>
  <tr>
    <td><code>emailVerifiedMatchError</code></td>
    <td>There's a mismatch with the email, or the user didn't verify their email.</td>
    <td>Update <code>email</code>, or send user new [authorization URL](../../../accounts/memberships/index.mdx#invite) to verify email.</td>
  </tr>
  <tr>
    <td><code>idVerifiedMatchError</code></td>
    <td>The user wasn't assigned the right identification level.</td>
    <td>The user needs to complete another [identification process](../../../users/identifications/index.mdx).</td>
  </tr>
</table>
