1. On your Swan Dashboard, go to **Settings** > **Onboarding**.
1. In the onboarding URLs section, toggle **Public URLs** to **on**.
1. Notice the URLs that appear for individuals and companies for each account country.

:::caution Public URL availability
Anyone could create an account using the public URL, and all accounts created are charged to your project.
:::

:::caution Public URLs toggle and the onboarding URL
With the current onboarding API, the `onboardingUrl` is only returned in the create or update mutation response when the **Public URLs** toggle is enabled. If you create onboardings with the API and let your users complete them through Swan's onboarding form, enable this toggle before going live. You can find it under **Settings** > **Onboarding** in the onboarding URLs section.
:::
