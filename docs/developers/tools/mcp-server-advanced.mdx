# Advanced usage

The MCP server tools support optional parameters that fine-tune how schema information is retrieved and formatted. Most developers won't need these settings, but they're available for specialized use cases.

## search-graphql-schema parameters

### keywords

**Required**. Array of search terms to find relevant schema parts.

**Type**: `string[]`

**Example**:

```json
{
  "keywords": ["payment", "transaction", "SEPA"]
}
```

### limit

Maximum number of search results to return.

**Type**: `number`  
**Default**: `20`  
**Range**: `1-50`

**Example**:

```json
{
  "keywords": ["account"],
  "limit": 10
}
```

### contextSize

Maximum size of the returned schema snippet in characters.

**Type**: `number`  
**Default**: `10000`  
**Range**: `1000-50000`

Use smaller values for simpler queries or larger values when you need comprehensive context.

**Example**:

```json
{
  "keywords": ["mutation", "credit transfer"],
  "contextSize": 15000
}
```

### includeSearchResults

Include individual search results with relevance scores in the response.

**Type**: `boolean`  
**Default**: `false`

Useful for debugging search behavior or understanding why specific types were included.

**Example**:

```json
{
  "keywords": ["card"],
  "includeSearchResults": true
}
```

## introspect-graphql-type parameters

### types

**Required**. Names of GraphQL types to introspect.

**Type**: `string[]`

You can introspect multiple types simultaneously.

**Example**:

```json
{
  "types": ["Account", "Payment", "Transaction"]
}
```

### includeNested

Include related types and nested fields in the response.

**Type**: `boolean`  
**Default**: `true`

Set to `false` to get only the specified types without their dependencies.

**Example**:

```json
{
  "types": ["Account"],
  "includeNested": false
}
```

### maxDepth

Maximum depth for exploring nested type relationships.

**Type**: `number`  
**Default**: `3`  
**Range**: `1-5`

Higher values provide more complete context but return larger responses.

**Example**:

```json
{
  "types": ["Payment"],
  "maxDepth": 2
}
```

### contextSize

Maximum size of the returned schema snippet in characters.

**Type**: `number`  
**Default**: `15000`  
**Range**: `1000-50000`

**Example**:

```json
{
  "types": ["Mutation"],
  "contextSize": 20000
}
```

## Usage patterns

### Focused type inspection

Get just the core definition without dependencies:

```json
{
  "types": ["AccountMembership"],
  "includeNested": false,
  "maxDepth": 1,
  "contextSize": 5000
}
```

### Deep relationship exploration

Explore complex type hierarchies:

```json
{
  "types": ["Mutation"],
  "includeNested": true,
  "maxDepth": 4,
  "contextSize": 30000
}
```

### Broad search with details

Cast a wide net for search results:

```json
{
  "keywords": ["direct debit", "mandate", "SEPA"],
  "limit": 30,
  "contextSize": 20000,
  "includeSearchResults": true
}
```

## Response size optimization

The MCP server returns compact, relevant schema portions instead of the full 200KB+ schema. However, you can optimize further:

**For simple queries**: Use smaller `contextSize` values (5000-10000 characters) and `maxDepth` of 1-2.

**For complex integrations**: Use larger `contextSize` values (20000-30000 characters) and `maxDepth` of 3-4.

**For debugging**: Enable `includeSearchResults` to understand search behavior.

## Best practices

1. **Start with defaults**: The default parameters work well for most use cases.
2. **Adjust contextSize based on your AI model**: Smaller models benefit from smaller context sizes.
3. **Use includeNested selectively**: Disable it when you only need type definitions without dependencies.
4. **Limit search results**: Keep `limit` values reasonable (10-20) for better response times.
5. **Iterate depth as needed**: Start with `maxDepth: 2` and increase only if you need deeper relationship exploration.
