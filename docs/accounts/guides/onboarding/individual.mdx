## Individual onboarding form {#onboarding-form}

Through the individual onboarding form, Swan collects the following information:

1. User's **email address**.
1. User's **country of residence** and **residence address**.
1. Basic details about the **user's employment** and **financial situation**. This information allows Swan to perform two essential tasks:
    * During onboarding, Swan completes an identity analysis (Know Your Customer, or KYC).
    * For the lifetime of the account, Swan monitors for fraud.
1. User [**signs up**](/topics/users/#signup) for Swan.

## Identification recommendations {#identification-recommendations}

Swan must verify the identity of every user.
Swan supports multiple [identification processes](/topics/users/identifications/#levels-processes), and some require a [first transfer](/accounts/guides/onboarding/account-holders#first-transfer).
For individual onboarding, the following levels are recommended based on the account country.

| Account country | Recommended level<br/>for individuals | Other supported levels |
|---|---|---|
| 🇫🇷 France | **<Yes /> PVID** | *Expert + first transfer<br/>QES* |
| 🇧🇪 Belgium | **<Yes /> QES** | *Expert + first transfer* |
| 🇩🇪 Germany | **<Yes /> QES + first transfer** | *none* |
| 🇮🇹 Italy | **<Yes /> QES** | *Expert + first transfer* |
| 🇳🇱 Netherlands | **<Yes /> Expert** | *QES<br/>PVID* |
| 🇪🇸 Spain | **<Yes /> QES** | *Expert + first transfer<br/>PVID + first transfer* |

### Unique links using the API

Create a unique individual onboarding link for each user with the API.

1. Call the `createIndividualAccountHolderOnboarding` mutation.
1. Enter information for all required API fields for the account country, as noted in [country requirements for individual accounts](#country-reqs).
1. Include optional fields as needed for your use case (such as `accountInfo.name` or `oAuthRedirectParameters`).
1. Add optional messages to the success payload, either for validation or in case of rejection.

The previous [`onboardIndividualAccountHolder`](https://api-reference.swan.io/mutations/onboard-individual-account-holder/) mutation is deprecated.

:::

### Guide

Update an individual onboarding by **adding an account name**.

1. First, [retrieve the ID](/accounts/guides/onboarding/#get-info) for the onboarding you need to update.
1. Call the `updateIndividualAccountHolderOnboarding` mutation.
1. Enter the onboarding ID retrieved in step 1.
1. Add the `accountInfo.name` field with an account name (example: Personal account).
1. Add desired messages to the success payload, either for validation or in case of rejection.

The previous `updateIndividualOnboarding` mutation is deprecated.

Use `updateIndividualAccountHolderOnboarding` for all new integrations.
